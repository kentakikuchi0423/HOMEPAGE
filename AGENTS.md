# AGENTS.md — プロジェクトルール

## プロジェクト概要

政治活動用の1ページ完結プロフィールサイト。
有権者に候補者のプロフィール・政策・X投稿・寄付導線を届けることが目的。

## 技術スタック

| 項目 | 採用技術 |
|---|---|
| フレームワーク | Next.js (App Router, static export) |
| スタイリング | Tailwind CSS |
| アニメーション | Framer Motion または CSS transitions |
| ホスティング | Netlify |
| 開発環境 | Dev Container (Docker) |
| 言語 | TypeScript |

## 絶対ルール

**コンテンツの変更は `src/content/site.ts` のみ編集する。**

- テキスト・URL・設定値をハードコードしない
- コンポーネントに文字列を直書きしない
- `site.ts` の型定義（`SiteConfig`）を変更する場合は `docs/architecture.md` も更新する

## デザイン方針

- **カラー**: 青ベース（`primary` カラー系）
- **トーン**: 親しみやすさと信頼感の中間
- **スタイル**: ポップで洗練されている
- **動き**: スクロール連動のフェードインなど、適度なアニメーション
- **レスポンシブ**: モバイルファーストで設計

## フェーズ制約

| Phase | 作業内容 | 許可される操作 |
|---|---|---|
| 0 | ドキュメント整備 | ドキュメントファイルのみ作成 |
| 1 | プロジェクト初期化 | `npx create-next-app`, 設定ファイル作成 |
| 2 | レイアウト実装 | 共通コンポーネント作成 |
| 3 | セクション実装 | 各セクションコンポーネント作成 |
| 4 | スタイル仕上げ | アニメーション・デザインの調整 |
| 5 | デプロイ設定 | `netlify.toml`, `next.config.ts` の設定 |
| 6 | QA | Lighthouse 計測・モバイル確認・コンテンツ調整 |
| 7 | 本番公開 | Netlify へのデプロイ実行 |

## 初期スコープ外（実装禁止）

以下は初期スコープ外とする。将来的な追加は別フェーズで検討。

- Supabase / Firebase などの BaaS
- データベース（DB）全般
- 認証・ログイン機能
- 管理画面・CMS
- 問い合わせフォーム（メール送信機能）
- 寄付決済処理（外部サービスに委譲）

## コマンド早見表

```bash
# 開発サーバー起動（dev container 内）
npm run dev

# 本番ビルド（静的エクスポート）
npm run build

# ビルド成果物の確認
npx serve out
```
